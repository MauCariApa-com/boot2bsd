# Contributing to Boot2BSD

Thank you for your interest in contributing to the Boot2BSD project! As a simulation of a vintage PowerPC Unix environment, we value contributions that enhance the authenticity and functionality of the system.

## How to Contribute

### 1. Reporting Bugs
If you find a command that doesn't behave like a traditional BSD utility or encounter a visual glitch in the CRT simulation, please open an issue with:
- A clear description of the bug.
- Steps to reproduce the issue in the terminal.
- Expected vs. actual behavior.

### 2. Feature Requests
We are always looking to expand our simulated toolset. If you'd like to see a specific Unix utility implemented (e.g., `awk`, `top`, or `network` tools), feel free to suggest it.

### 3. Code Contributions
- **Style**: Maintain the TypeScript and React patterns used in the project.
- **Authenticity**: Ensure new commands match the behavior of FreeBSD 2.x or 3.x utilities.
- **Documentation**: Every new command must include a corresponding manual page in `src/constants.ts`.

### 4. Documentation
Improving the `man` pages or the system's `motd` (Message of the Day) is a great way to contribute to the project's lore and usability.

## Contributor Covenant
By participating in this project, you agree to abide by our code of conduct, maintaining a respectful and collaborative environment for all vintage computing enthusiasts.
